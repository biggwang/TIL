## 프로세스 커뮤니케이션 (IPC - Inter Process Communication)

### 프로세스A가 직접 프로세스B를 접근 할 수 없다.

### 근데 프로세스간 통신이 왜 필요할까??
- 요즘 CPU는 멀티코어
- 성능(속도)을 높이기 위해 여러 프로세스를 만들어서 동시 실행
- 이 때 프로세스간 상태 확인 및 데이터 송수신이 필요 한것이다.


### 실제 프로세스에서 커널 공간(물리메모리)은 공유한다. (리눅스 예)
즉 IPC, 프로세스간 통신하는 방법은 바로 이 커널 공간을 이용하여 통신한다는 의미

### IPC 기법
- Message Queue
- Shared Memory
- Pipe
- Signal
- Semaphore
- Socket

결국 프로세스간 공유자원인 커널영역을 공유하는 기법

### 프로세스 각 영역 메모리 할당 구조
- 프로세스당 4GB나 되지만 실제 물리주소에만 올라가는건 일부분만 올라가므로 더 적음
- 0~3Gb(stack, heap, data, text 등) user 영역
- 3Gb~4Gb kernel 영역
- 프로세스간 공유는 바로 위 kernel 영역(물리메모리에 할당됨)을 공유한다.

### Pipe
C언어에서 fork()라는 시스템 콜을 예를 들면 fork() 시스템콜을 사용하는 순간 부모프로세스가 자식프로세스로 그대로 복사하여 독립적인 프로세스를 만든다. 
그러면 **커널영역**에서 pipe라는 기술을 이용하여 단방형으로 부모 -> 자식으로 프로세스간 공유해야 할 데이터가 이동된다.

### 공유메모리
커널영역에 노골적으로 공유 메모리 공간을 만들고 key값을 이용하여 프로세스간 데이터를 공유 할 수 있는 영역

### Signal
유닉스에서 30년이상 사용된 전통적인 기법으로 커널 또는 프로세스에서 다른 프로세스에 어떤 이벤트가 발생 되었는지를 알려주는 기법 말그데로 이벤트!

주요 시그널:기본동작  
- SIGKILL: 프로세스 종료
- SIGALARM: 알랄 발생  
등등

### Socket
client - server 네트워크 통신 할 때 사용하는 기술이지만 하나의 컴퓨터 안에서 프로세스간 통신 할 때도 사용함
