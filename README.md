
# :book: 개발 일지 
[![Version](https://img.shields.io/badge/version-2018.12.23-red.svg)]()  

에피파니(Epiphany) 적인 경험, 반성, 개념, Snippet 등 머릿속 추상적인 것들이나 배워야 할 내용을 글로 배출하면서 체화시켜 어제보다 발전한 나를 만드는 도구

# :lock: 작성규칙
- 모든 개발 관련 지식 최초 Input은 TIL에서 먼저 작성
- 이해한 내용을 바탕으로 **나만의 언어** 로 풀어서 작성
- 못한것은 주말에 **반드시** 마무리
- 제목에 반드시 TAG 부터 붙인다.

  - **[NEW]** : TIL 작성만 된 지식
  - **[PLAN]** : 실천 계획을 만들고 실습을 통하여 나의 무기로 삼고 싶을 때
  - **[POSTING]** : 작성한 주제를 좀 더 공부하여 Posting 하고 싶을 때
  - **[APPLIYED]** : 실무 적용 해보았을 때
