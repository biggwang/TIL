인터페이스를 만들고 그 구현 객체 A,B가 있다고 하였을 때, 구현 객체 A를 B로 바꾸싶으면 
인터페이스 변수에 넣은 것을 B객체로만 바꿔 주면 된다.  

만약 인터페이스를 선언하지 않고 객체 A를 B로 바꾸게 되면 A와 B가 일관성이 없어서 B클래스에 맞게 소스 코드 수정을 해야 할 것이다.  

또 한가지 기능은 메소드 파라미터를 인터페이스 변수로 하면 객체 다형성을 활용을 극대화 할 수 있다.  
인터페이스를 구현한 객체가 여러개 있을때 함수 파라미터에 그 넣고 싶은 구현 객체를 넣어 주면 다양하게 객체 기능을 스위칭 하여 활용 할 수 있기 때문이다.
