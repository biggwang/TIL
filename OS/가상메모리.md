### 가상 메모리
실제 각 프로세스마다 4GB를 물리 메모리에 할당하는 것은 한계가 있음 그래서 필요한 개념  
여러 프로세스를 동시에 실행 할 때 필요함

실제는 4GB가 모두 물리메모리에 올라가지 않고 일부만 물리메모리에 올라간다.  

### 기본 아이디어
프로세스는 가상 주소를 사용하고, 실제 해당 주소에서 데이터를 읽고/쓸대만 물리 주소로 바꿔주는 개념  

- Virtual address (가상주소): 프로세스가 참조하는 주소
- Physical address (물리주소): 실제 메모리 주소

가상주소 -> 물리주소 변환해주는 것이 시간이 좀 걸림 그래서 나온 개념이 MMU

### MMU(Memory Management Unit)
CPU에 코드 실행시, 가상 주소 메모리 접근이 필요할 때, 해당 주소를 물리 주소값으로 변환해주는 하드웨어 장치

# CPU, 가상메모리, MNU 관계
CPU는 가상메모리를 요청을 MNU가 받고 MNU 장치가 물리 메모리에 접근 (하드웨어 장치를 이용해야 주소변환이 빠르므로)
