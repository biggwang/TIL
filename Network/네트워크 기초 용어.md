### 스위치

이더넷 네트워크(사설망)에서 스위치라는 기기에서 컴퓨터 끼리 연결을 한다.  
스위치는 MAC 주소 테이블 정보를 가지고 있는데 포트 번화와 출발지 MAC 주소 맵핑정보를 가지고 들어온 요청을 각 PC에 전달 한다.

### 라우팅

이더넷으로 만든 네트워크와 네트워크를 연결 할 때 사용하는 네트워크 기기이다.  
라우팅 테이블을 이용하여 해당 목적지에 패킷을 전송한다. 라우팅 테이블에는 목적지 네트워크 IP정보가 있다.

### ARP

MAC주소와 IP 주소 서로 호환을 위해 ARP를 사용함  

### NAT(Network Address Translation)

사설망 IP 주소와 공인망 IP 주소를 일대일로 연결하여 변환해주는 역활  


### TIL(Time To Live)

패킷이 네트워크망을 거치면서 얼마동안 살아 있어야하는지 나타내는 수치 보통 8bit로 255까지 값이 있으며 라우터를 거치면서 1씩 감소한다.  
DNS Casching 에서 TTL은 Contents DNS에서 받아온 값(IP)를 얼마나 보관할지에 대한 개념이다.  
값을 너무 짧게 Cashing 하면 DNS가 장애가 났을때 값이 날라가 서비스 장애가 올 수 있다.


### 클러스터링

여러대의 서버를 네트워크로 연결하여 전체적으로 하나의 서버처럼 보이게 하는 기술
각 서버는 하트비트로 서로를 감시하며 1번서버가 죽게되면 사용자가 요청한 리퀘스트 다른 서버로 옮기면서 끊기지 않고 응답할 수 있게 한다.

### 서버 부하분산 기술 1 - DNS 라운드로빈 

DNS서버에서 도메인에 대응하는 여러 IP 맵핑 테이블을 만들어 놓고 통신을 배분하여 서버 부하를 분산 시킨다.

### 서버 부하분산 기술 2 - 서버타입

서버에서 인스톨한 소프트웨어를 이용하는 방법이다. 예로 Ngnix Proxy를 이용하여 부하 분산을 시킨다.

### 서버 부하분산 기술 3 - 어플라이언스 서버 타입

부하분산용 서버를 이용하는 방법이다. 아마 로브벨런서 같은 녀석을 말하는거 같다.

### 광역 부하분산 기술

부하분산을 위한 기술이라기보다 재해 발생하였을 때 다른 지역에 서버에서 서비스를 계속 제공하기 위함이다.


### IDS(Instrusion Detection System), IPS(Instrusion Prevention System)

IDS는 악의적인 요청에 대한 감지를 하는 시스템이다. 공격패턴을 기억해 놓고 있다가 패턴이 맞아 떨어지면 관리자에게 경고를 보냄
IPS는 IDS에서 받을 알림을 가지고 실제 차단을 시키는 시스템이다.

### UPS 

정전이 일어나거나 전원이 차단 되었을 때 자체 배터리로 서버가 안정적으로 종료 될 수 있도록 하고, 번개로 과전압이 흘러 들어오면 전원을 차단하는 역활도 한다.
