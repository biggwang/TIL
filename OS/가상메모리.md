### 가상 메모리
실제 각 프로세스마다 4GB를 물리 메모리에 할당하는 것은 한계가 있음 그래서 필요한 개념  
여러 프로세스를 동시에 실행 할 때 필요함  

웹페이지에서 페이징 개념을 연상하면 쉽게 이해 됨. 수천만개 로우가 있다고 했을때 전부 불러오는 것은 무리가 있으므로 페징으로 일정 갯수가 조회 하여 필요한 데이터만 가져오는 개념을 생각하면 된다.

실제는 4GB가 모두 물리메모리에 올라가지 않고 일부만 물리메모리에 올라간다.  

### 기본 아이디어
프로세스는 가상 주소를 사용하고, 실제 해당 주소에서 데이터를 읽고/쓸대만 물리 주소로 바꿔주는 개념  

- Virtual address (가상주소): 프로세스가 참조하는 주소
- Physical address (물리주소): 실제 메모리 주소

가상주소 -> 물리주소 변환해주는 것이 시간이 좀 걸림 그래서 나온 개념이 MMU

### MMU(Memory Management Unit)
CPU에 코드 실행시, 가상 주소 메모리 접근이 필요할 때, 해당 주소를 물리 주소값으로 변환해주는 하드웨어 장치

### CPU, 가상메모리, MNU 관계
CPU는 가상메모리를 요청을 MNU가 받고 MNU 장치가 물리 메모리에 접근 (하드웨어 장치를 이용해야 주소변환이 빠르므로)


### 페이징 시스템
- 각 프로세스마다 일정 부분을 물리 메모리에 할당 하게 되는데 그 일정부분을 얼마만큼에 사이즈로 할당 할것인가에 대한 개념
- 각 프로세스(4GB)의 PCB에 Page Table 구조체를 가리키는 주소가 들어 있음
- 프로세스 안에는 가상메모리에 Page(리눅스 4KB)가 나누어져 있고 Page Table에는 가상메모리와 실제 물리메모리 맵핑정보를 가지고 있음
- 프로세스가 4GB를 사용하는 이유 - 32bit 시스템에서 2의 32제곱이 4GB

### Page Table
가상 주소 v = (p, d)  
- p: 페이지 번호
- d: 페이지 처음부터 얼마 떨어진 위치인지

해당 프로세스에서 특정 가상 주소 엑세스를 하려면 해당 프로세스의 PCB에 page table의 Base 주소 + page 번호를 확인 후  
page 번호가 있으면 이 page가 매핑된 첫 물리 주소를 알아내고(p`) p'+d 실제 물리 주소가 된다.
